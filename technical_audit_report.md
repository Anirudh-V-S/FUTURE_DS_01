## 8. Required Modifications Before Moving to Power BI
Before importing the cleaned dataset (`data/processed/cleaned_superstore.csv`) into Power BI, you must implement these minor schema adjustments:

1.  **Create a Dedicated Calendar Dimension (`Dim_Calendar`):**
    *   *Issue:* Retail sales often have date gaps (e.g., no orders placed on Christmas Day). If you map Time Intelligence directly to `order_date`, calculations like YoY growth will return blank or incorrect figures.
    *   *Fix:* Build a continuous, gap-free Date Table in Power BI using DAX:
        ```dax
        Dim_Calendar = 
        CALENDAR(
            MIN(cleaned_superstore[order_date]), 
            MAX(cleaned_superstore[order_date])
        )
        ```
2.  **Explicit Data Modeling Relationships:**
    *   In Power BI, hide the raw dimension columns (Category, Region, Customer Name) inside the Fact Table, and build a clean Star Schema linking your fact columns to dedicated Dimension Tables for a clean data model.
3.  **Standardize Currency Settings:**
    *   Explicitly format `Sales`, `Profit`, and `Average Order Value` as **Currency (USD, $)** and set decimal points to 2. By default, Power BI may read them as generic decimal numbers.
